### 4. Удаление пользователей (новое)

**Файлы:** `src/pages/admin/AdminUsers.jsx`

**Логика:**
```javascript
const canDeleteUser = (u) => {
  if (u.id === currentId) return false; // себя удалять нельзя
  if (u.role === "super_admin") return currentRole === "super_admin"; // super_admin только суперадмин
  if (u.role === "admin") return currentRole === "super_admin"; // admin только суперадмин
  return true; // кандидатов и модераторов может админ и суперадмин
};

const deleteUser = async (u) => {
  // Soft delete: статус → blocked
  await base44.entities.ReferralProfile.update(u.id, { status: "blocked" });
  // Логируем в ActionLog
  await base44.entities.ActionLog.create({...});
};
```

**Результат:**
✅ Админ может удалять кандидатов и модераторов  
✅ Суперадмин может удалять админов  
✅ Используется soft delete (status: blocked), не hard delete  
✅ Логируется в ActionLog  
✅ Подтверждение перед удалением

### 6. Region Code автоматизация

**Файлы:** `src/lib/regionHelpers.js` (новый файл)

**Реализация:**
```javascript
export function guessRegionCode(input) {
  // Справочник регионов: "москва" → "MSK"
  // Автоматически заполняется при вводе названия региона
  return REGION_CODES[normalized] || input.slice(0, 3).toUpperCase();
}
```

**Справочник включает:**
- ✅ Все крупные города и область РФ
- ✅ Альтернативные названия (МО, Свердловская область)
- ✅ Fallback: первые 3 буквы в верхнем регистре

**Результат:**
✅ При вводе "Москва" → автоматически "MSK"  
✅ При вводе "Краснодар" → автоматически "KRD"  
✅ Админ может переопределить вручную  
✅ Справочник полный для основных регионов

## 🚀 ЧТО ОСТАЛОСЬ НА ФАЗУ 2

### Модератор: полноценный кабинет
- Отдельная вкладка "Программы" с детальным обзором
- Аналитика по программам, кандидатам, выплатам
- Инструменты роста дерева (подсвет слабых узлов, возможности расширения)

### Дерево программ: интерактивное
- Раскрытие кандидатов в узлах дерева
- Быстрые переходы между уровнями
- Визуализация эффективности ветвей

### Выплаты: полный workflow
- Колонка "Выплаты" в списке кандидатов
- Статус выплат (pending → approved → paid)
- Breakdown цепочки распределения
- Аналитика по выплатам

### Multi-program: через membership
- Полная поддержка ProgramMembership в UI
- Портфель программ с фильтрацией
- Роли участия (owner, participant, moderator)

### Логирование и безопасность
- Полное логирование всех действий в ActionLog
- История изменений статусов программ
- Аудит доступа к секретным кодам
