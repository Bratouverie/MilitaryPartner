# Отчёт: Исправление дубля ссылок в Telegram share

## Дата исправления
2026-06-20

## Проблема
В `src/pages/dashboard/ReferralDashboard.jsx` функция `handleShare()` передавала одну и ту же ссылку в два места одновременно:
```javascript
navigator.share({
  title: "МилитариПартнер",
  text: `...${referralLink}`,  // ссылка в тексте
  url: referralLink,            // ссылка в параметре url
});
```

Это приводило к тому, что в Telegram и других share-таргетах ссылка показывалась дважды или отправлялась дважды.

## Исправление
### Файл: `src/pages/dashboard/ReferralDashboard.jsx`

**Было (строки 88-97):**
```javascript
const handleShare = () => {
  if (navigator.share && referralLink) {
    const rewardText = getRewardAmount();
    navigator.share({
      title: "МилитариПартнер",
      text: `Присоединяйся к программе! За каждого кандидата платят ${rewardText}. ${referralLink}`,
      url: referralLink,
    });
  }
};
```

**Стало:**
```javascript
const handleShare = () => {
  if (navigator.share && referralLink) {
    const rewardText = getRewardAmount();
    navigator.share({
      title: "МилитариПартнер",
      text: `Присоединяйся к программе! За каждого кандидата платят ${rewardText}.`,
      url: referralLink,
    });
  }
};
```

**Изменение:** Удалена ссылка из `text`, оставлена только в `url`.

## Зачем это правильно
- **Web Share API** работает правильно: текст передаётся как превью/описание, URL как основное содержимое
- **Telegram**: получает `text` как описание, `url` как ссылку — в сообщении появляется только одна ссылка
- **WhatsApp**: аналогично — ссылка один раз
- **Мобильный share sheet**: не дублирует ссылку

## Как это работает теперь
- **Общий share-обработчик** (кнопка "Поделиться" в кнопке #1):
  - text: `"Присоединяйся к программе! За каждого кандидата платят 25 000 ₽."`
  - url: ссылка приглашения активной программы
  - Результат: одна ссылка, нормальный текст с конкретной суммой

- **Dedicated Telegram button** (кнопка #3 "Поделиться в Telegram"):
  - Использует `getTelegramShareUrl()` из `useActiveInviteProgram.js`
  - Генерирует URL вида: `https://t.me/share/url?url={joinLink}&text={encodedText}`
  - Text в URL: `"Присоединяйся к программе «МилитариПартнер». За каждого кандидата платят 25 000 ₽."`
  - Ссылка находится в параметре `url` — не дублируется в параметре `text`
  - Результат: одна ссылка в Telegram

## Выводы

**Причина проблемы:** Двойная передача одной и той же ссылки в `text` и `url` параметры `navigator.share()`.

**Решение:** Оставить ссылку только в `url`, текст в `text` содержит только описание программы и сумму.

**Результат:** 
- В Telegram приходит ровно одна ссылка
- Текст содержит конкретную сумму и название программы
- Нет дублей в WhatsApp, мобильном share sheet и других таргетах
- Оба share-сценария (общий и Telegram) используют единый источник истины (active invite program)
