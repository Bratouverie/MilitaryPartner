# Отчёт: Исправление логики трёх больших кнопок на Dashboard реферала

## Дата исправления
2026-06-20

## Проблемы до исправления

1. **Жёсткий хардкод диапазона суммы**: "50 000 – 200 000 ₽" использовался во всех местах вместо конкретной суммы из active invite program
2. **Вторая кнопка не переключалась**: она проверяла `profile.email` вместо реального статуса PaymentProfile
3. **Dashboard и Payouts-flow не связаны**: не было механизма отслеживания заполнения payout данных
4. **Telegram share с плохим текстом**: содержал диапазон "от 50 000 до 200 000" вместо конкретной суммы

## Изменённые файлы

### 1. `src/lib/payoutHelpers.js` (новый файл)
**Что добавлено:**
- `isPayoutProfileComplete(paymentProfile)` — проверка, заполнены ли обязательные поля (ФИО, дата рождения, паспорт, адрес)
- `formatRewardAmount(amount)` — форматирование суммы по-русски с пробелами и символом ₽
- `generateTelegramShareText(inviteProgram)` — генерация текста Telegram share с конкретной суммой

**Зачем:**
- Единый источник истины для определения completeness payout профиля
- Централизованное форматирование сумм
- Правильный текст для share без хардкода

### 2. `src/lib/useActiveInviteProgram.js` (обновлён)
**Изменения:**
- Импорт `generateTelegramShareText` и `formatRewardAmount` из payoutHelpers
- Добавлен метод `getRewardAmount()` — возвращает отформатированную сумму из `inviteProgram.reward_quota`
- Добавлен метод `getCandidateLink()` — возвращает ссылку на форму кандидата из `candidate_form_code`
- Обновлён метод `getTelegramShareUrl()` — теперь использует конкретную сумму из программы

**Что это значит:**
- Хук предоставляет не только ссылку, но и отформатированную сумму
- Хук генерирует и ссылку на партнёра, и ссылку на кандидата
- Единая логика для всех компонентов (ReferralDashboard, MyLink, Overview)

## Как работает новая логика

### Источник истины для активной программы приглашения
```
useActiveInviteProgram(profileId)
├── inviteProgram (объект с reward_quota, candidate_form_code, и т.д.)
├── inviteLink (ссылка на join: /join/{link_code})
├── getRewardAmount() → форматированная сумма из reward_quota
├── getCandidateLink() → ссылка на форму кандидата
└── getTelegramShareUrl() → Telegram share с конкретной суммой
```

### Определение "данные заполнены"
```javascript
isPayoutProfileComplete(paymentProfile) 
  ✓ recipient_name заполнено
  ✓ date_of_birth заполнено
  ✓ passport_series заполнено
  ✓ passport_number заполнено
  ✓ registration_address заполнено
```

### Поток кнопок
```
Пользователь заходит на dashboard
  ↓
ReferralDashboard загружает activeInviteProgram + PaymentProfile
  ↓
Кнопка #1: Показывает конкретную сумму из reward_quota
  ↓
Кнопка #2: 
  если payoutComplete = false → "Заполнить данные для выплаты"
  если payoutComplete = true → "Анкета кандидата" (жёлтая, 3 действия)
  ↓
Кнопка #3: Telegram share с точной суммой
```
